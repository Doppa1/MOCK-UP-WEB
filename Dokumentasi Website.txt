1. PENDAHULUAN
   1. Nama Website : Website Desa Rasau Jaya Tiga
   2. Tujuan/Fungsi Website
Website ini dibuat sebagai sarana digital resmi pemerintah Desa Rasau Jaya Tiga untuk:
      1. Menyampaikan informasi dan berita terkini kepada warga desa.
      2. Menyediakan layanan administrasi desa secara online.
      3. Menjadi media publikasi kegiatan, potensi, dan struktur pemerintahan desa.
      4. Mendorong partisipasi masyarakat melalui transparansi dan keterbukaan informasi.
   3. Deskripsi Singkat
Website ini dikembangkan dengan menggunakan framework frontend modern berbasis ReactJS dan Tailwind CSS. Struktur halaman dibuat modular agar mudah dikembangkan dan diintegrasikan dengan backend pada masa mendatang. Website ini mencakup halaman-halaman penting seperti Beranda, Profil Desa (Sejarah, Geografis, Visi-Misi, Struktur), Galeri, Berita, Layanan Publik, dan Kontak. Dokumentasi ini berisi panduan teknis bagi pengembang dan administrator untuk pengelolaan dan pengembangan fitur lebih lanjut.

4. PENJELASAN KOMPONEN
   1. Header.jsx
      1. Fungsi: Navigasi utama website (desktop & mobile)
      2. Detail Kode:
         1. Menggunakan useState untuk menyimpan status buka/tutup menu mobile.
         2. Navigasi utama menggunakan <Link> dari react-router-dom agar tidak reload halaman.
         3. Dropdown untuk menu Profil Desa dan Pemerintahan menggunakan struktur div.relative.group dengan hover untuk membuka menu.
         4. Tombol hamburger mobile membuka menu navigasi vertikal di bawah header.
   2. Footer.jsx
      1. Fungsi: Menampilkan informasi dasar dan penutup halaman
      2. Detail Kode:
         1. Dibagi menjadi beberapa kolom (grid md:grid-cols-5).
         2. Kolom 1: deskripsi desa + ikon media sosial (fab fa-facebook-f, dll).
         3. Kolom 2: tautan ke instansi pemerintah (hardcoded list).
         4. Kolom 3: kontak desa.
         5. Kolom 4: embed Google Maps menggunakan iframe.
         6. Statistik dan informasi kredit di bagian bawah.
   3. MainContent.jsx
      1. Fungsi: Konten utama halaman beranda
      2. Detail Kode:
         1. Section 1: Video latar belakang videoDesa (autoplay, muted, loop).
         2. Section 2: 4 layanan utama ditampilkan melalui .map() dari array inline.
         3. Section 3: Berita terbaru hardcoded satu card.
         4. Section 4: Agenda dan pengumuman (hardcoded list).
         5. Section 5: Sambutan Kepala Desa (gambar + teks statis).
         6. Section 6: Profil Pegawai (4 pegawai dummy di-grid .map())
         7. Section 7: Google MyMaps embed iframe untuk fasilitas umum.
   4. Beranda.jsx
      1. Fungsi: Halaman utama website
      2. Detail Kode:
         1. Mengimpor MainContent, Header, dan Footer, tetapi hanya MainContent yang dirender.
   5. Galeri.jsx
      1. Fungsi: Menampilkan dokumentasi kegiatan desa
      2. Detail Kode:
         1. Menggunakan array fotoGaleri[] untuk menyimpan data galeri (id, src, judul).
         2. .map() digunakan untuk membuat grid galeri responsif 3 kolom.
         3. Gambar diambil dari unsplash dengan keyword pencarian.
   6. Kontak.jsx
      1. Fungsi: Halaman untuk menghubungi desa
      2. Detail Kode:
         1. Hero section (judul dan breadcrumb).
         2. Kolom kiri: Formulir kontak (nama, email, pesan) dengan HTML input dan textarea.
         3. Kolom kanan: Informasi kontak dan jam layanan.
   7. Layanan.jsx
      1. Fungsi: Menampilkan daftar layanan administrasi desa
      2. Detail Kode:
         1. Data layanan disimpan dalam array layananList[], lalu ditampilkan dengan .map() sebagai card layanan.
         2. Sidebar: fitur pencarian (belum aktif), informasi tambahan dan kontak admin.
   8. Sejarah.jsx
      1. Fungsi: Menampilkan sejarah desa
      2. Detail Kode:
         1. Tampilan gambar logo dan ilustrasi sejarah.
         2. Paragraf panjang sejarah dalam elemen <p> dengan text-justify.
         3. Sidebar berisi: cari berita, berita terbaru, dan data desa.
   9. Struktur Desa.jsx
      1. Fungsi: Menampilkan tabel struktur organisasi pemerintahan desa
      2. Detail Kode:
         1. Tabel terdiri dari jabatan dan nama (array inline berisi pasangan jabatan-nama).
         2. Sidebar sama seperti Sejarah.jsx

6. KEAMANAN
Hal-hal yang Perlu Dipersiapkan untuk Keamanan Digital Website
Keamanan digital sangat penting agar data desa tidak disalahgunakan dan situs tetap aman dari serangan. Berikut hal-hal yang perlu dipersiapkan:
Aspek Keamanan Digital - Rencana & Tindakan Pencegahan

No  | Aspek Keamanan Digital       | Tindakan yang Direkomendasikan
----+------------------------------+---------------------------------------------------------------------------
1   | HTTPS & Sertifikat SSL       | Pastikan domain menggunakan HTTPS dengan sertifikat SSL agar data dienkripsi.
2   | Autentikasi Admin            | Gunakan sistem login aman dengan proteksi password dan token (JWT atau session).
3   | Validasi Input Form          | Selalu validasi input dari pengguna untuk mencegah XSS, SQL Injection, dan spam.
4   | Rate Limiting & Captcha      | Gunakan batasan percobaan login dan CAPTCHA untuk mencegah brute-force dan bot.
5   | Backup Rutin                 | Lakukan backup otomatis database dan file minimal seminggu sekali.
6   | Update Dependencies          | Periksa dan perbarui dependensi (npm packages) secara berkala untuk mencegah kerentanan.
7   | Role-based Access Control    | Batasi fitur yang bisa diakses oleh pengguna dengan role tertentu (admin, editor, viewer).
8   | Audit Log                    | Simpan log aktivitas admin untuk keperluan monitoring dan keamanan.
9   | Firewall & Hosting Aman      | Gunakan layanan hosting dengan proteksi DDoS dan firewall bawaan.
10  | Training Admin               | Edukasi admin desa soal phishing, password kuat, dan penggunaan dashboard yang aman.
	



Area Rawan Kerentanan & Jenis Serangan Umum
Analisis Kerentanan dan Solusi Mitigasi Keamanan

Area / Komponen         | Potensi Kerentanan            | Jenis Serangan Umum                | Solusi atau Mitigasi
------------------------+-------------------------------+------------------------------------+------------------------------------------------------------
Form Kontak (Kontak.jsx)| Input tidak divalidasi        | XSS, Email Injection, Spam Bot     | Validasi input, gunakan reCAPTCHA, filter karakter berbahaya
Login Admin (dev)       | Autentikasi tidak aman        | Brute Force, Credential Stuffing   | Hash password, gunakan 2FA/JWT, rate limiting
Upload Gambar (Galeri)  | Upload file tanpa pengecekan  | File Malware, Overwrite, DOS       | Validasi ekstensi & ukuran file, gunakan file scanner
Backend API             | Endpoint tanpa otorisasi      | Unauthorized Access, Data Leakage  | Middleware auth, validasi token, batasi rate akses
Dependensi Lama         | Library tidak diperbarui      | Exploitable Vulnerability          | Gunakan npm audit, update rutin
URL Routing             | Jalur admin terekspos         | URL Guessing, Path Traversal       | Proteksi autentikasi untuk halaman sensitif
Hosting / Database      | Server default atau terbuka   | Data breach, defacing, ransomware  | Hosting aman, backup terenkripsi, nonaktifkan default ports
